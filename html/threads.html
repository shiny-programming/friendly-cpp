threads

In C++, threads basically lets you to perform multiple tasks simultaneosly and efficiently


Quick Info Box:


Header: <thread>

Type: Concurrency

Best for: Running tasks in parallel






How it works


Normally, a program runs step by step:

Task 1 → Task 2 → Task 3


With threads, you can run tasks in parallel:

Task 1⎤
"""""""├ → running at the same time
Task 2⎦
Task 3 (also running)


So instead of waiting for one task to finish, you split work into smaller parts that run concurrently.

Here is an example for further comprehension-



Example:

#include <thread>;
#include <iostream>;

void printHello(){
,,,,,,,std::cout << "Hello from thread! \n";
}

int main(){
,,,,,,,std::thread t(printHello);

,,,,,,,t.join();

,,,,,,,std::cout << "Main function done \n"
,,,,,,,return 0;
}

Output

Hello from thread!
// The thread t starts and runs printHello()

// t.join() makes the main function wait

// So the thread finishes first

Main function done
// Then the "Main function done"





Common Operations


  • std::thread() → Creates a new thread(worker) that runs task() separately


    Main Thread → starts → [New Thread running task()]


    Example:""std::thread t(task);




  • join() → Main thread waits until the worker thread is done


    Worker Therad ↴ (finishes)
    """"""""""""""""""""├-- waiting...
    Main Thread ⎽⎽⏌

    Then main continues


    Example:""t.join();




  • detach() → Run threads independently


    Main Thread --------▻ continues immediatly

    Worker Thread --------▻ runs on its own (independent)


    • Thread runs in the background.
    • You can't control it anymore after detaching

    Example:""t.detach();





  • get_id() → Returns the id of the thread (useful for debugging and tracking threads)


    Thread A → ID: 101

    Thread B → ID: 102


    Example:""std::cout << t.get_id();





Real Use Case


Threads are very popular in real-world projects.
It is used when your program needs to do multiple things at the same time without slowing down.
Some use case:


  1. Games:

    Game loop:

    - Render grapics

    - Handle player input

    - Play sound


    How threads help -

    Thread 1 → Grapics(rendering)

    Thread 2 → Keyboard/Mouse input

    Thread 3 → Sound/music


    Result: Smooth gameplay (no lag when doing multiple tasks).




  2. Websites / Apps:

    You click "Login"


    Behind the scenes -

    Thread 1 → UI (button, animation)

    Thread 2 → Send request to server

    Thread 3 → Load data


    Result: App doesn't freeze while loading.




  3. Downloading Files:

    Downloading a large file


    With threads -

    Thread 1 → Download file

    Thread 2 → Show progress bar

    Thread 3 → Let user browse app


    Result: You can still use the app while downloading.




  4. Image / Video Processing:

    Editing a photo or rendering video


    With threads -

    Thread 1 → Process top half

    Thread 2 → Process bottom half


    Result: Much faster processing.




  5. Chat Application:

    You use Messenger / WhatsApp


    With threads -

    Thread 1 → Send message

    Thread 2 → Receive messages

    Thread 3 → Update UI


    Result: Real-time chatting without delay.




  6. Servers(Very Important) :

    Multiple users accessing a website


    In this case threads -

    User A - Threads 1

    User B - Threads 2

    User C - Threads 3


    Result: Many users can use the service at the same time.




Important Note

Always use join() or detach().

Failing to do so can crash your program.
