mutex

A synchronization tool that prevents multiple threads from accessing shared data at the same time.


Quick Info Box:


Header: <mutex>

Type: Thread Synchronization (Concurrency)

Best for:Protecting shared data from race conditions






How it works


Thread A -----▻ 🔒 Locked Resource
'''''''''''''''''''''''''''''''''''''|
'''''''''''''''''''''''''''''''''''''|
Thread B -----▻ ⏳ Waiting

When Thread A finishes:
🔓 Unlock

Thread B -----▻ Access Resource


A mutex acts like a key to a room.

  • Only one thread can hold the key.

  • Other threads must wait.

  • This prevents data corruption.

Here is an example for further comprehension-



Example:

#include <map>;
#include <iostream>;
#include <thread>;

std::mutex mtx;

void printMessage()
{
,,,,,,,mtx.lock();

,,,,,,,std::cout << "Hello from thread\n";

,,,,,,,mtx.unlock();

}

int main(){
,,,,,,,std::thread t1(printMessage);
,,,,,,,std::thread t2(printMessage);

,,,,,,,t1.join();
,,,,,,,t2.join();

}

Output

Hello from thread // Thread 1
Hello from thread // Thread 2

// Thread 1 --▻ Print --▻ Done
'''''''''''''''''''''''''''''''''''''''''''''''''''''''''''''↓
// Thread 2 --▻ Print --▻ Done






Common Operations


  • lock() → Locks the mutex.

    If another thread already owns the mutex, the current thread must wait.



    Thread A """ Thread B
    '''''' | ''''''''''''''''''''''''' |
    '''''.├─lock() ''''''' |
    '''''' | '''''''''''''''''''''''''├─lock()
    '''''' | ''''''''''''''''''''''''' |
    '''''' | Working'''''' | Waiting...
    '''''' | ''''''''''''''''''''''''' |
    ''''' └─ unlock(). |
    ''''''''''''''''''''''''''''''''''. |
    '''''''''''''''''''''''''''''''''. └─ Gets the lock


    Example:""mtx.lock();




  • unlock() → Releases the mutex so another thread can use it.


    Mutex: [LOCKED]
    ''''''''''''''''''''''''''↓
    '''''''''''''''''''unlock()
    ''''''''''''''''''''''''''↓
    Mutex: [FREE]


    Example:""mtx.unlock();




  • try_lock() → Attempts to lock the mutex.

    Unlike lock(), it does not wait.



    Resource Free?
    '''''|
    Yes --▻ Lock
    No --▻ Continue


    Code:

    if (mtx.try_lock())
    {
    // Got the lock
    mtx.unlock();
    }





  • std::lock_guard → Automatically locks and unlocks a mutex.

    This is the most beginner-friendly and safest option.



    Function Starts
    ''''''''''''''''''↓
    lock_guard created
    ''''''''''''''''''↓
    Mutex Locked
    ''''''''''''''''''↓
    Do Work
    ''''''''''''''''''↓
    Function Ends
    ''''''''''''''''''↓
    Mutex Automatically Unlocked


    Example:""std::lock_guard<std::mutex> lock(mtx);


    Code:

    void updateScore()
    {
    std::lock_guard<std::mutex> lock(mtx);
    score++;
    }





  • std::unique_lock→ A more flexible version of lock_guard.

    It can lock and unlock multiple times.



    unique_lock created
    ''''''''''''''''''↓
    Lock
    ''''''''''''''''''↓
    Do Some Work
    ''''''''''''''''''↓
    Unlock Early
    ''''''''''''''''''↓
    Do Other Work


    Example:""std::unique_lock<std::mutex> lock(mtx);





Real Use Case


Mutex is useful whenever multiple threads need to access the same data safely.

Without a mutex, threads can interfere with each other and produce incorrect results.
A mutex makes sure only one thread modifies shared data at a time.
Some use case:


  1. Banking App:

    Imagine two users transferring money at the same time.


    Without Mutex -

    Account Balance = $1000

    Thread A: Withdraw $100
    Thread B: Withdraw $200

    Both read $1000

    Result: Wrong Balance ❌


    With Mutex -

    Thread A
    ''''''''''|
    '''''''''├─ Lock Account
    '''''''''├─ Withdraw $100
    '''''''''└─ Unlock


    Thread B
    ''''''''''|
    '''''''''├─ Lock Account
    '''''''''├─ Withdraw $200
    '''''''''└─ Unlock


    Result: Correct Balance ✅


    Result: Prevents money calculations from becoming incorrect.




  2. Multiplayer Game:

    Many players may interact with the same game world simultaneously.


    Without Mutex -

    Player A ──┐
    ''''''''''''''''''''''''''''├─ Shared Game Data
    Player B''' ─┤
    '''''''''''''''''''''''''''''|
    Player C''''─┘


    With Mutex -

    Player A → Update Position
    '''''''''''''''''''''''''''''''''''↓
    ''''''''''''''''''''''''''Mutex Lock

    Player B → Wait

    Player C → Wait


    Result: Prevents player stats, inventories, and scores from becoming corrupted.




  3. Logging System:

    Applications often have multiple threads writing to the same log file.


    Without Mutex -

    User LoginSucDatabase Connectedcess


    With Mutex -

    User Login Success

    Database Connected


    Result: Keeps log files readable and organized.




  4. Web Server Visitor Counter:

    A website may track how many visitors have viewed a page.


    Without Mutex -

    Visitor 1 ─┐
    Visitor 2 ─┼─ Counter++
    Visitor 3 ─┘

    Expected: 1000 visitors
    Actual: 997 visitors ❌


    With Mutex -

    Lock Counter

    Increase Count

    Unlock Counter


    Result: Ensures every visit is counted correctly.




  5. Download Manager:

    Several files may download simultaneously.


    Without Mutex -

    Download 1 ─┐
    Download 2 ─┼─ Total Progress
    Download 3 ─┘


    With Mutex -

    Download 1 updates progress
    ''''''''''''''''''''''''↓
    Download 2 updates progress
    ''''''''''''''''''''''''↓
    Download 3 updates progress


    Result: Keeps progress percentages accurate.




  6. Factory Production System:

    Imagine multiple machines updating the same inventory database.


    Without Mutex -

    Machine A ─┐
    Machine B ─┼─ Inventory Stock
    Machine C ─┘


    With Mutex -

    One machine updates stock

    at a time


    Result: Prevents incorrect inventory counts.




Important Note


A mutex is used to prevent two threads from modifying the same data, whcih might produce unexpected result.

When to Use and Not to Use Mutex:

✅ Shared variables

✅ Shared files

✅ Shared resources

✅ Multiple threads writing data

❌ Not needed when threads work independently
