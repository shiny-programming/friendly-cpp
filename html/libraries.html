Libraries


C++ provides a variety of built-in libraries that helps the developers to code easiliy and efficiently.

In this section, you'll learn about three important catagories: STL(Standard Template Library), Concurrency tools, Utilities libraries.

Each of this catagories and libraries in it will be explained in a simple(almost simple) and practical way so you can actually use them in real-world projects, simplify common programming task and improve performance.



STL(Standard Template Library)


The STL is a part of C++ that provides generic (Template-based) classes and functions. Think of it as a ready-made toolbox in C++.

Instead of building everything from scratch( and pulling your hairs out :D ), STL gives you powerful tested components like vectors, stacks and maps etc. to store data and work with it efficiently. In other words, you can use the same structure for different data types (int, flloat, string, etc.).

Some important and popular components are:

  • vectors
  • map
  • unordered-map
  • set




Concurrency


Concurrency simply means doing multiple tasks at the same time. In C++, this multi-tasking ability was first added in C++ 11 to improve the performance of the programme, it's response time and obviously multi-tasking.

Instead of programme doing:

Task A → then Task B → then Task C



It can do:

Task A + Task B + Task C (at the same time)

There are 4 essential parts in Concurrency that programmers use the most :

  • Threads
  • Mutex
  • Lock Guards
  • Future & Async




Utilities


Utilities components are like small helper tools that don't store data (like containers) or process data (like algorithms), but instead help you manage and organize your code better and makes your code safer. Although, this one is a bit overlooked by programmers but it is super useful once you start writing real code.
