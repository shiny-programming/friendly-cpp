optional

A container that may contain a value or may contain nothing.


Quick Info Box:


Header: <optional>

Type: Utility (C++17)

Best for:Returning values that might not exist






How it works


Imagine a treasure box:

┌───────────┐
│'''''''' Value......''''' │
└───────────┘

OR

┌───────────┐
│'''''''' Empty....'''''' │
└───────────┘


An optional can either:

✅ Hold a value

or

✅ Be empty

std::optional<int>

Case 1:
┌───────────┐
│''''''''''' 42.........''''' │
└───────────┘
Contains a Value

OR

Case 2:
┌───────────┐
│'''''''' Empty....'''''' │
└───────────┘
Contains nothing


Here is an example for further comprehension-



Example:

#include <optional>;
#include <iostream>;

std::mutex mtx;

int main()
{
,,,,,,,std::optional<int> age = 18;

,,,,,,,if(age.has_value())
,,,,,,,{
,,,,,,,,,,,,,,std::cout << *age;
,,,,,,,)

}

Output

18


// In this case the container "age" has value and that is 18(integer)
┌───────────┐
│''''''''''''''' 18...........'' │
└───────────┘






Common Operations


  • Create Optional → Create an optional value.


    optional<int>
    ''''''''''↓
    ''''''''18


    Example:""std::optional<int> num = 18;




  • has_value() → Check if a value exists.


    [18] --▻ true
    [ ] --▻ false


    Example:""num.has_value();




  • value() → Get the stored value.


    [18]
    ' ↓
    [18


    Example:""num.value();




  • value_or() → Use a default value if the container is empty.


    Empty
    ''''''''↓
    Default Value


    Example:""num.value_or(0);




  • reset()→ Remove the stored value.


    Before: [18]
    ''''''''↓
    reset()
    ''''''''↓
    After: [ ]


    Example:""num.reset();




  • emplace()→ Create/Replace a Value


    Before: [42]
    ''''''''↓
    emplace(18)
    ''''''''↓
    After: [18]


    Example:""num.emplace(18);





Real Use Case


Optional is useful whenever something may exist or may not exist.

Instead of using fake values like -1,"Unknown" or nullptr you can clearly say:
"This value is optional"

Some use case:


  1. User Profile Nickname:

    Not every user has a nickname.


    User Profile A

    Name: Alex
    Nickname: lex


    User Profile B

    Name: Alvin
    Nickname: (None)


    Without std::optional, you'd need special values like:


    nickname = "";


    Which can be confusing.



  2. Login System:

    Searching for a user by ID:


    Searching User #42...

    +-----------------+
    | '' Not Found ''|
    +-----------------+


    or,

    Searching User #42...

    +-----------------+
    | ''''' Found ''''''''|
    +-----------------+


    The fuunction can return:


    return user;

    or

    return std::nullopt;




  3. Weather Application:

    Sometimes weather data is unavailable..


    Today's Temperature

    +-----------------+
    | ''''''' 29°C '''''''',|
    +-----------------+


    or,

    Today's Temperature

    +--------------------------+
    | '''Data Unavailable''''|
    +--------------------------+






  4. Game Development:

    A player may or may not have a target selected.


    Player

    '''''↓

    [ Goblin ]


    Or,

    Player

    No target selected


    No need for fake IDs like:


    targetId = -1;




  5. Database Fields:

    Some users provide a phone number.

    Some don't.


    User Record A

    Name: Alice
    Phone: 123456789


    User Record B

    Name: Jame
    Phone: Not Provided






  6. Configuration Settings:

    A game may allow custom FPS limits.


    Settings

    FPS Limit: 144


    or,

    Settings

    FPS Limit: default





  7. API Responses

    An online service may not always return every field.


    {
    '''''''"username": "Eric",
    '''''''"bio": "Loves C++",
    }


    or,

    {
    ''''''''"username": "Troy",
    '''''''"bio": null,
    }






Important Note


⚠ Check Before Using

Bad:

std::optional<int> num;

std::cout << num.value();


This can throw an exception because the optional is empty.

Good:

if(num.has_value())
{
'''''std::cout << num.value();
}



When to Use and Not to Use Optional:

✅ Search results

✅ User input

✅ Database lookups

✅ Functions that may fail

✅ Replacing special values like -1

❌ Not for values that must always exist



Why Optional Is Better Than Returning -1


Without optional

Age Found → 25

Age Not Found → -1


But what if -1 becomes meaningful?


With optional:

Age Found
''''''''''↓
optional<int>(25)


Age Not Found
''''''''''↓
empty optional


Much clearer and safer.
