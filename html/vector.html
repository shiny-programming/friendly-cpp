vector

A Dynamic array or a smart list that can grow and shrink automatically as your program runs.


Quick Info Box:


Header: <vector>

Type: Container(STL)

Best for: Dynamic arrays






How it works


Vector grows automatically when new elements are added. It also shrinks automatically when an element is removed.

Here is an example for further comprehension-



Example:

#include <vector>;
#include <iostream>;

int main(){
,,,,,,,std::vector<int> v = {1, 2, 3};
,,,,,,,v.push_back(4);

,,,,,,,for(int i : v){
,,,,,,,,,,,,,,cout << i << " ";
,,,,,,,}
,,,,,,,std::cout << v.size();
,,,,,,,return 0;
}

Output

1, 2, 3, 4
// added an element (4) at the end.
4
// size of the vector.





Common Operations


  • push_back() → Add an element at the end of the vector


    [1] [2] [3] → push_back(4) → [1] [2] [3] [4]




  • pop_back() → Remove last element


    [1] [2] [3] [4] → v.pop_back(4) → [1] [2] [3]




  • size() → Return the number of elements


    [1] [2] [3] [4] → v.size() → 4




  • clear() → Remove all the elements


    [1] [2] [3] [4] → v.clear() → [] (empty vector)





Real Use Case


Vector is commonly used in real-world projects.
It is very useful when you need a list of things where the size is not fixed.
Some use case:


  1. User Inputs (Most Common):

    When the user types numbers or inputs and you don't know how many they'll enter.
    Vectors can be very efficient in these cases, since it can grow automatically.




  2. Game Development:

    In games, you constantly need to add or remove objects:

    Enemies:

    [ ] → [Enemy1] → [Enemy1 Enemy2] → [Enemy1 Enemy2 Enemy3]

    Enemy dies:

    [Enemy1 Enemy2 Enemy3] → [Enemy1 Enemy3]



    👉 Perfect for:
    • Enemies
    • Bullets (when Firing or Reloading)
    • Particles
    • Hearts (health system in a player or npcs)



  3. To-Do List / App Data:

    In this type of application people have to add or remove data.
    Vector can be a big help in this case, as it is dynamic.

    User adds tasks:

    Tasks:
    [ ] → ["Study"] → ["Study", "Code"] → ["Study", "Code", "Sleep"]



    User deletes one:

    Tasks:
    ["Study", "Code", "Sleep"] → ["Study", "Sleep"]





  4. Grapics/OpenGL:

    You can store a number of vertices in a vector, which will create different types of shapes in the window

    Triangle(2D): [(x1, y1), (x2, y2), (x3, y3)]

    Add more details:
    → [more points added]


    In this way, you do not have to hardcode size.
    you can build graphical objects dynamically




  5. Scores/ Data Tracking:

    Vector can be efficiently used for tracking data or recording scores in a game.

    Scores:
    [100] → [100 200] → [100 200 300]



    👉 Great for:
    • history
    • logs
    • Stats



Performance

  • Fast insertion at the end
  • Slow insertion in the middle


Common Mistakes

Sometimes programmers tend to make mistakes while using vectors.
While programming be careful of these mistakes ↴
  • Forgetting that inserting in the middle is slow
  • Invalidating iterators after resizing
